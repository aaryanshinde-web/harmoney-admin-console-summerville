# Edit My Profile & Help Docs

> Your admin identity, your personal activity log, your password, and the help link-out.

## Step-by-Step Workflow

### Step 1 — My Profile

First Name, Last Name, E-Mail Id, Username, Mobile, Role, Enrolled Since. Only Mobile is self-maintained — Role and E-Mail are locked for segregation of duties.

![Step 1 — My Profile](/tmp/_profile_help_md/img/ep-01-my-profile.jpg)

### Step 2 — My Operations

Pick an operation from Choose operation, set Start Date and End Date, Search. Personal slice of Admin Activity — the evidence for control-review attestations.

![Step 2 — My Operations](/tmp/_profile_help_md/img/ep-02-my-operations.jpg)

### Step 3 — Change Password

New Password and Confirm Password, each with a Show toggle. Password Requirements panel alongside (8–25 chars, one numeric, one lower, one upper).

![Step 3 — Change Password](/tmp/_profile_help_md/img/ep-03-change-password.jpg)

### Step 4 — Help Docs

Link-out to Product / Content documentation. Deliberately not an editable module — its value is being in the same console the admin already works in.

## Summary

Two personal surfaces. Profile has three tabs — record, activity log, password. Help Docs routes out to the bank's published documentation.

## Key Use Cases

- First login → My Profile, verify identity, set Mobile.
- Quarterly access review → My Operations, date-bounded search, evidence for sign-off.
- 90-day rotation → Change Password, satisfy policy, submit.
- Terms & Conditions question mid-change → Help Docs in-console lookup, no context switch.
